## 📘 项目复盘笔记：NanoChat-SFT 微调实践

### 一、关键经验总结

#### 1. 数据清洗与准备
- **原始数据格式**：JSON 日志嵌套 HTML，需用 BeautifulSoup 解析提取纯文本对话。
- **关键点**：确保每轮对话的“用户-助手”配对正确，避免错位。
- **特殊 token 处理**：在 `tiktoken` 编码时，必须显式允许 `<|endoftext|>` 特殊 token，否则会报错。
- **损失掩码（mask）**：SFT 的核心——只对助手回复部分计算损失。需要记录每个样本中 response 的起始位置，生成对应的 mask 数组。
- **数据量评估**：3000+ 条对话对足够让模型学会角色风格，但需配合 LoRA 或小学习率防止过拟合。

#### 2. 环境配置与依赖
- **虚拟环境**：务必使用 `venv` 或 `conda` 隔离项目，避免依赖冲突。
- **网络问题**：国内下载预训练模型或库时，配置镜像（`HF_ENDPOINT=https://hf-mirror.com`）或使用代理。
- **WSL 路径**：Windows 文件挂载在 `/mnt/c/`，但训练应在 WSL 原生文件系统（`/home/...`）中进行，避免性能问题和权限错误。

#### 3. 训练调试
- **学习率**：小模型（<10M）初始学习率不宜超过 `3e-4`，否则容易梯度爆炸（loss 变为 `nan`）。出现 `nan` 时先降低学习率，并检查 mask 分母是否为 0。
- **梯度裁剪**：设置 `grad_clip=1.0` 或更低，防止梯度爆炸。
- **mask 检查**：训练初期打印每个 batch 的 `mask.sum()`，确保不为 0，否则 loss 无法计算。
- **设备选择**：CPU 训练极慢但可用于验证流程；GPU 训练需注意显存，调小 `batch_size` 和 `block_size`。
- **预训练权重**：若网络不通，可先用随机初始化模型验证流程，后续再加载预训练权重。

#### 4. Git 与 GitHub 操作
- **敏感信息**：**绝对不要提交包含 API Key、token 的文件**（如 `chat.json`）。GitHub 会自动检测并阻止推送。
- **补救措施**：若误提交敏感文件，需从 Git 历史中彻底移除（`git rm --cached` + `amend` 或 `filter-branch`），然后强制推送。
- **分支名**：GitHub 默认分支为 `main`，本地可能是 `master`，推送前需重命名或指定分支。
- **网络不稳定**：HTTPS 推送失败时，改用 SSH 协议（配置 SSH key）更稳定。
- **忽略文件**：及时创建 `.gitignore`，排除虚拟环境、二进制数据、敏感文件等。

#### 5. 工程化实践
- **代码组织**：将清洗、准备、训练脚本分离，保持模块化。
- **文档**：编写清晰的 README，包含项目背景、使用步骤、注意事项，并附上训练日志作为佐证。
- **版本控制**：用 Git 管理代码，每次修改提交清晰的信息，便于回溯。

### 二、遇到的问题与解决方案

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| `pip install` 慢或失败 | 国内网络问题 | 使用清华源 `-i https://pypi.tuna.tsinghua.edu.cn/simple` |
| Hugging Face 下载超时 | 无法访问国外网站 | 设置 `HF_ENDPOINT=https://hf-mirror.com` 镜像 |
| 脚本缩进错误（TabError） | 混用空格和制表符 | 统一使用空格，或在编辑器中设置“将制表符转换为空格” |
| 编码时特殊 token 报错 | `tiktoken` 默认禁止特殊 token | 添加参数 `allowed_special={"<|endoftext|>"}` |
| loss 变成 `nan` | 学习率过高 / mask 全零 | 降低学习率，打印 mask 总和检查分母 |
| 训练时 mask 全零 | 数据中 response 为空或长度不足 | 过滤过短样本，或在数据准备时检查 token 长度 |
| Git 推送被拒绝（含 secret） | 提交了包含 token 的文件 | 从缓存移除文件，修改最后一次提交，强制推送 |
| Git 推送网络错误（GnuTLS recv） | HTTPS 不稳定 | 改用 SSH 协议，或设置代理 |
| 本地分支名与远程不一致 | 本地默认 `master`，远程 `main` | 重命名本地分支 `git branch -m master main` |
| 训练时找不到文件 | 路径错误或未生成二进制文件 | 确认 `data/sft/` 下存在 `train.bin` 等文件，检查脚本路径 |

### 三、后续优化方向

- **大规模训练**：在云端（AutoDL）用 GPU 训练更多步数（如 5000 步），观察 loss 下降至 4~5，并生成角色对话示例。
- **LoRA 微调**：引入 LoRA 技术，减少参数量，防止过拟合，同时加快训练。
- **对话界面**：用 Gradio 或 Streamlit 搭建简易 Web UI，输入 prompt 即可看到模型回复，增强展示效果。
- **效果对比**：记录不同训练步数下的生成文本，分析模型学习过程。
- **多角色分离**：尝试为每个角色训练独立的 LoRA 插件，或引入角色 embedding，提升角色一致性。
